# 1-Use ADR Format

* Status: Accepted
* Date: 2026-02-05

## Context and Problem Statement

We need a standard way to document architectural decisions in the project.

## Decision Drivers


* Standardization

* Documentation

* Traceability


## Considered Options


* ADR (Markdown)

* Wiki

* Code Comments


## Decision Outcome

Chosen option: "ADR (Markdown)", because ADRs are standard, easy to read in the repo, and versioned with the code.

### Consequences

* Good: Better transparency and historical context.
* Bad: Slightly more overhead when making decisions.
